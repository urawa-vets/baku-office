# Google 連携（サービスアカウント・キーレス）の自動セットアップ

この手順は **Google Cloud Shell**（ブラウザ内のターミナル・ログイン済み）で実行します。
GCP プロジェクト作成・API有効化・サービスアカウント作成・**Workload Identity Federation（鍵レス）**の
設定までを自動で行い、最後に baku-office の設定画面に貼る値を表示します。所要 2〜3 分です。
**サービスアカウント鍵(JSON)は作成しません**（組織ポリシーで鍵作成が禁止されていても動きます）。

## 始める前に

- この Cloud Shell は、いま開いているあなたの Google アカウントとして動きます。
- プロジェクト作成や API 有効化の権限（と、必要なら課金アカウント）があるアカウントで開いてください。

最初に「承認（Authorize）」を求められたら **Authorize** を押してください。

## ステップ 1：セットアップを実行する

baku-office の設定画面で **「② コマンドをコピーして Cloud Shell を開く」** を押すと、**実行コマンドはすでにコピー済み**です。あとは **3つだけ**：

1. **「Authorize（承認）」** をクリック。
2. ターミナルをクリックして **Ctrl+V（Mac は Cmd+V）** で貼り付け。
3. **「Enter」キー**を押す ← これで実行が始まります（貼り付けただけでは実行されません）。

> 💡 もしコピーされていなかった場合は、baku-office の設定画面の「実行コマンド」欄の **「コピー」ボタン**を押してから、ここで貼り付けてください。コマンド先頭の `WORKER_URL=` は、その欄であなたのURLが自動設定されています（手で書き換える必要はありません）。

参考：貼り付くコマンドはこの形です（URL とトークンは自動で埋め込まれます）。

```sh
WORKER_URL=https://（あなたのURL） HANDOFF=（自動） bash google-service-account-setup.sh
```

あとは**入力不要で自動進行**します（代理ユーザーはログイン中のアカウントを自動採用。API反映の待ちは
自動リトライ）。完了すると、画面の最後に **WIF設定（1行 JSON）・クライアントID・スコープ** がまとまります。

> 別のユーザーを代理させたい場合のみ、`SUBJECT=メール` を先頭に足して実行してください
> （通常は不要。最終的には baku-office の設定画面の欄が優先されます）。

## ステップ 2：baku-office に戻って「登録して接続テスト」

> ✅ 完了時、**WIF設定は自動で baku-office に送信**されます（設定画面の URL から実行した場合）。タブに戻ると **WIF設定欄が自動入力**されているので、コピーは不要です。

baku-office の「Google連携セットアップ → サービスアカウント」画面で：

1. **WIF設定（JSON）** … 自動入力されています。**もし入っていない場合のみ**、出力の「┌─ … └─」の間の **1行 JSON** をコピーして入力欄に貼り付け（Ctrl+V / Cmd+V）。
2. **代理するユーザー** … 既定で団体アカウントが入っています（変える場合のみ修正）
3. **利用する機能** を選んで **「登録して接続テスト」** を押す

## ステップ 3：ドメイン全体の委任を承認する（1回だけ）

「登録して接続テスト」の後に表示される **クライアントID** と **スコープ** を、
Google Workspace 管理コンソールの「ドメイン全体の委任」に登録します。

- 管理コンソール：<https://admin.google.com/ac/owl/domainwidedelegation>
- ここだけは Google がセキュリティ上、人による承認を必須にしているため自動化できません。

## 完了

承認したら、baku-office で **もう一度「登録して接続テスト」** を押してください。
成功すれば連携完了です（このタブは閉じてかまいません）。
